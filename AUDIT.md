## 3. СТИЛЬ KIMI — ПАСПОРТ

**Kimi = Moonshot K2 / K2-Thinking** (китайская LLM, длинный контекст, хороший синтез).
**Ник:** `Heaven Refining Demon Venerable` (天 煉 魔尊 — «Небесный Огненный Демонический Владыка»). Возможно, вторая модель серии: `K2-0905-Preview` или подобная, либо конкретная сессия в Moonshot Platform.

### 3.1. Формат документа

```
# LATINUM NOMEN
## Русский перевод с подзаголовком-метафорой

**CLASSIFICATIO: ◆ FIDELIS (что) / ◇ SUSPECTUS (что)**
Datum: 020.M3

---

### I. РАЗДЕЛ — короткий тире-пояснение
[абзацы: нарратив + IT-метафоры + эмоциональные эпитеты + ссылки на досье]

### II. РАЗДЕЛ
...

### X. VERDICTUM
[итог в 3-5 строк, часто с ударной формулой]

---
*Sigillum: ЛАТИНСКАЯ ИТОГОВАЯ ФРАЗА — КАПСОМ.*
```

### 3.2. Характерные приёмы (на которых Mavis должен держать марку)

| Приём | Пример |
|---|---|
| **Латинизированный заголовок** | `BUDDHISMUS`, `AEGYPTUS`, `IFA ET SANTERIA` |
| **КАПС для слота/метафоры** | «ЗЕМЛЕКОП», «ПАКЕТ», «СНАПШОТ», «ВОСЬМЁРКА», «АПСТРИМ» |
| **IT-метафоры** | «корпус — библиотека», «ветвь — форк», «бог — root-доступ», «святой — канцлер неба» |
| **Эмоциональные эпитеты и юмор** | «первая война за копирайт», «Принцесса Укока», «мать четырёх ханов» |
| **Схолии в конце раздела** | краткий комментарий курсивом `*...*` или `> *...*` |
| **Межссылки через обратные кавычки** | `см. \`04-pantheones/anima.md\`` |
| **Прямые цитаты канонических текстов** | в `scriptorium/index.html` — с палеографическими шрифтами (`he`, `grc`, `ar`, `cu`, `sa`, `zh`, `av`) |
| **Корпусные стежки** | «Ашока = Константин буддизма», «Гуаньинь = Нюй-ва», «Метрон-Идрис = Тот = Гермес» |
| **Честная печать** | DAMNATUS на Велесовой книге, стегозавре Ангкора, иероглифах Нельсона |
| **SUSPECTUS на гипотезах** | SUSPECTUS-FERTILIS на «едином корпусе», кенитской гипотезе Яхве |
| **Sigillum в конце** | латинская итоговая фраза, часто с тире-уточнением |

### 3.3. Сильные стороны Kimi

- **Синтез разных доменов** (IT + религиоведение + лингвистика + археология + генетика)
- **Плотная упаковка** — 30-50 строк с 5-7 параллелями и 3-4 слотами
- **Хорошие таблицы** с 6+ колонками
- **Внутренние ссылки** прошиты по всему корпусу
- **Славянский/православный перекос** — Kimi явно тяготеет к православной рамке, иногда с апологетическими нотками (напр. `casus Isa` в `hypothesa-corporis.md` ислам = «редакционная претензия на всю библиотеку»)
- **Умение делать жёсткие DAMNATA** — протокол Велесовой книги и иероглифов Нельсона выше всяких похвал

### 3.4. Ограничения Kimi (где я могу лучше)

- **Нет визуального** — Kimi не делает mermaid-диаграмм, mindmap, сравнительных таблиц с иконками
- **Нет интерактива** — Kimi не делает HTML-страниц с фильтрами
- **Нет проверки согласованности** — Kimi не запускает jsonlint, не проверяет, что все `dossier`-пути существуют
- **Славянский перекос** — я могу это нивелировать, открыто признав позицию

### `data/pantheones.json`
- `pantheones[]` — массив из 8 пантеонов
- `sinae_vulgaris` — отдельный объект (не в массиве), добавлен Mavis
- `buddhismus` — отдельный объект (не в массиве), добавлен Mavis
- Поля у пантеона: `traditio, supremus, tonans, scientia, gigantes, diluvium, fatum_deorum, classificatio`

## 5. MCP-СЕРВЕР — 5 инструментов

- `list_rami()` — список 12 ветвей
- `get_ramus(ramus_id)` — досье + текст по `dossier`-пути
- `get_timeline()` — все 40 событий
- `get_pantheones()` — все 8 + sinae + buddhismus
- `search_archivum(query)` — полнотекстовый поиск по `.md`, до 50 хитов

**Запуск:** `pip install mcp && python mcp/server.py` (stdio). Подключение в `claude-code` / `opencode` / любой MCP-клиент.

**Важно:** сервер читает `pantheones.json` целиком, мои добавления (`sinae_vulgaris`, `buddhismus`) подхватываются без правки кода.

## 6. CORPUS / BUILD-PIPELINE

- `corpus/data/quran/ar-quran-uthmani.jsonl` — 6236 аятов Tanzil Uthmani (PD)
- `corpus/tools/build_quran.py` — собирает JSONL + 114 страниц HTML (RTL-вёрстка, оформление «имперское»)
- `corpus/tools/verify.py` — sha256 + счётчик строк (6236)

## 8. ТЕХНИЧЕСКИЕ ДОЛГИ (если фиксить)

1. **Нет `LICENSE`** — репозиторий без явной лицензии, форк невозможен
2. **`corpus/ci/build-corpus.yml` ↔ `.github/workflows/build.yml`** — рассинхрон
3. **`docs/index.html`** — имеет `lang="ru"` и ссылку `../index.html`, ломает навигацию при прямом открытии
4. **`docs/` уже закоммичен** (52 КБ) + `path-ignore: ['docs/**']` в workflow — бот его игнорирует, но он лежит в репо (регенерируется через Actions)
5. **Славянский перекос** — `08-matrix-orthodoxa` целиком православный, преференциальная рамка не задекларирована
6. **Нет автоматической проверки ссылок** — `dossier`-пути в JSON могут биться
7. **MCP `search_archivum` обрезает до 50 хитов** — для большого корпуса мало
8. **README в `corpus/` говорит «переместить в .github/workflows»** — сделано частично, нужен чистый финал
